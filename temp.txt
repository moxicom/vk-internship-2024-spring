Спроектируй архитектуру базы данных для следующего задания. И как лучше мне создать бд для docker compose, чтобы это реализовать:
REST API для управления базы данных фильмов.
Имеются пользователи - обычный пользователь и администратор.
Приложение должно поддерживать следующие функции:
* добавление информации об актере (имя, пол, дата рождения),
* изменение информации об актере.
Возможно изменять любую информацию об актере, как частично, так и полностью:
*удаление информации об актере, добавление информации о фильме
При добавлении фильма указываются его название (1-150 символов), описание (не более 1000 символов), дата выпуска, рейтинг (0-10) и список актерев:
*изменение информации о фильме.
Возможно изменить любую информацию о фильме, как частично, так и полностью:
*удаление информации о фильме
* получение списка фильмов с возможностью сортировки по названию, рейтингу, по дате выпуска. По умолчанию используется сортировка по рейтингу (по убыванию).
* поиск фильма по фрагменту названия, по фрагменту имени актера,
* получение списка актеров, для каждого актера выдается также список фильмов с его участием.
